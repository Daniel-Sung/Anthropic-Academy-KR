모듈 1: 설치 및 시작하기

Claude Code란?

코드베이스를 읽고, 파일을 편집하고, 명령을 실행하고, 개발 도구와 통합되는 에이전틱 코딩 도구입니다. 터미널, IDE, 데스크톱 앱, 브라우저에서 사용 가능합니다.

설치 방법

// macOS, Linux, WSL
curl -fsSL https://claude.ai/install.sh | bash

// Windows PowerShell
irm https://claude.ai/install.ps1 | iex

// 프로젝트에서 시작
cd your-project
claude

핵심 개념

Claude Code는 에이전틱 루프로 작동합니다. 사용자 지시 → 코드 분석 → 계획 수립 → 파일 편집 → 명령 실행 → 검증의 반복입니다.

  • CLAUDE.md — 프로젝트 루트에 두는 지침 파일. 코딩 표준, 아키텍처 결정, 리뷰 체크리스트 등 설정

모듈 1: Cowork란?

Claude Cowork 개요

Cowork는 Claude가 사용자의 실제 파일, 폴더, 앱에서 직접 작업하는 기능입니다. 코드뿐 아니라 슬라이드, 스프레드시트, 계약서, 문서 등 일반 업무 파일을 읽고, 편집하고, 실제 결과물을 만들어냅니다.

  • 개발자가 아닌 일반 직장인도 사용 가능
  • 브라우저에서 직접 작업 수행
  • 멀티스텝 태스크를 자율적으로 처리

모듈 2: Cowork 태스크 루프

태스크 루프 이해

Cowork는 사용자의 지시를 받아 계획 → 실행 → 검증의 루프를 반복합니다. 중간에 확인이 필요하면 멈추고 물어봅니다.

  • 계획 — 작업을 분석하고 단계별 계획 수립
  • 실행 — 파일 읽기/편집, 웹 검색, 도구 사용 등
  • 검증 — 결과를 확인하고 필요시 수정
  • 체크포인트 — 중요 지점에서 사용자 확인 요청
간단한 태스크 실행"이 문서를 요약해줘" 같은 단일 작업 시도
멀티스텝 태스크 실행여러 단계가 필요한 작업을 지시하고 루프 관찰

Claude에게 좋은 프롬프트를 작성하는 것은 "신입 직원에게 업무를 설명"하는 것과 같습니다.

6가지 핵심 기법

  • 명확하고 직접적으로 — 구체적인 출력 형식과 제약 조건을 명시하세요. "황금률: 프롬프트를 동료에게 보여주고 따라할 수 있는지 확인하세요."
  • 맥락 제공 — 왜 이 지시를 하는지 동기를 설명하면 Claude가 더 잘 이해합니다. 예: "줄임표를 쓰지 마세요" → "TTS 엔진이 줄임표를 발음할 수 없으므로 쓰지 마세요"
  • 예시 활용 (Few-shot) — 3~5개의 좋은 예시를 <example> 태그로 제공하면 정확도가 크게 향상됩니다
  • XML 태그로 구조화 — <instructions>, <context>, <input> 등으로 프롬프트를 구조화하면 Claude가 더 정확히 파싱합니다
  • 역할 부여 — 시스템 프롬프트에서 역할을 설정하면 행동과 톤이 집중됩니다. 예: "You are a helpful coding assistant specializing in Python."
  • 긴 컨텍스트 처리 — 20K+ 토큰의 문서는 프롬프트 상단에 배치하고, 질문은 마지막에 넣으세요. 성능이 최대 30% 향상됩니다
팁: Claude에게 "~하지 마세요"보다 "~하세요"가 더 효과적입니다. 예: "마크다운을 쓰지 마세요" → "매끄러운 산문 문단으로 작성하세요"

🤖 에이전트 (Agents)
0/4

Anthropic의 핵심 철학

"가장 성공적인 에이전트는 복잡한 프레임워크가 아니라 단순하고 조합 가능한 패턴으로 만들어진다."

워크플로우 vs 에이전트

  • 워크플로우 — LLM이 미리 정해진 코드 경로를 따름. 예측 가능하고 안정적
  • 에이전트 — LLM이 스스로 프로세스와 도구 사용을 결정. 유연하지만 비용과 지연 시간 증가

5가지 에이전트 패턴

  • 프롬프트 체이닝 — 작업을 순차 단계로 분해. 각 단계의 출력이 다음 입력이 됨. 예: 마케팅 카피 생성 → 번역
  • 라우팅 — 입력을 분류해서 전문화된 하위 작업으로 분배. 예: CS 질문 유형별 다른 처리
  • 병렬화 — 여러 LLM 호출을 동시 실행. 섹셔닝(독립 하위 작업) 또는 투표(같은 작업 여러 번)
  • 오케스트레이터-워커 — 중앙 LLM이 작업을 동적으로 분해하고 워커에게 위임. 예: 멀티 파일 코드 수정
  • 평가자-최적화 — 한 LLM이 생성, 다른 LLM이 평가하고 피드백을 반복. 예: 번역 품질 개선
핵심 원칙 3가지: 1) 단순하게 2) 투명하게 (계획 과정을 보여주기) 3) 도구 인터페이스를 꼼꼼히 설계 (주니어 개발자 문서 수준으로)

4D 프레임워크

AI를 잘 사용하기 위한 4가지 핵심 역량:

Delegation(위임) → AI에게 뭘 맡길지 판단
Description(기술) → 원하는 것을 정확히 표현
Discernment(판별) → AI 결과물을 비판적으로 평가
Diligence(성실) → 책임감 있게 AI 사용
